system is built this way, and what we've learned — lives in the
**[../brain/](../brain/)** folder, so the repo and the AI tool co-evolve there.

[../brain/workflow.md](../brain/workflow.md).**

- [../brain/conventions.md](../brain/conventions.md) — code & repo conventions (do / don't)
- [../brain/architecture.md](../brain/architecture.md) — system design & module map
- [../brain/glossary.md](../brain/glossary.md) — domain vocabulary
- [../brain/decisions.md](../brain/decisions.md) — why decisions were made (append-only)
- [../brain/journal.md](../brain/journal.md) — changelog of notable changes (append-only)
- [../brain/open-questions.md](../brain/open-questions.md) — backlog & known unknowns

**The one rule:** if you make a decision, change how the project works, or learn
something durable — record it in the brain *in the same change*. See
[../brain/workflow.md](../brain/workflow.md#updating-the-brain).
